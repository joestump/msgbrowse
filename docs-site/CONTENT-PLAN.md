## Where product docs go

Author product documentation under **`docs-site/docs/<section>/...`** as
Markdown/MDX. On every build, `scripts/copy-product-docs.js` copies this tree
into `docs-generated/docs/` (which is gitignored and wiped each build) so it is
served by the same docs plugin as the generated content.

- **Never** write into `docs-generated/` — it is regenerated by
  `scripts/build-docs.js` and anything you put there is deleted.
- `docs-site/docs/index.md` is the section landing page (route `/docs/`);
  replace or extend it.
- Routes: `docs-site/docs/getting-started/install.md` →
  `/msgbrowse/docs/getting-started/install`.

## How the sidebar works

`sidebars.ts` defines three autogenerated sidebars:

- `docsSidebar` — autogenerates from the `docs/` folder (your content). Add a
  `_category_.json` per section folder to control label/position, e.g.:

  ```json
  { "label": "Getting Started", "position": 1 }
  ```

  Use `sidebar_position` frontmatter for ordering pages within a section.
- `specsSidebar` / `decisionsSidebar` — the generated Architecture section
  (OpenSpec specs at `docs/openspec/specs/`, ADRs at `docs/adr/` in the repo
  root). Do not hand-edit; change the source ADRs/specs instead.

Navbar tabs: **Docs**, **ADRs**, **Specifications** (wired in
`docusaurus.config.ts`).

## Build commands

From `docs-site/`:

```sh
npm install        # once
npm run build      # runs scripts/build-docs.js (transforms + copy) then `docusaurus build`
npm start          # build content once + dev server
npm run serve      # serve the production build locally
```

The site is configured for GitHub Pages at `https://joestump.github.io/msgbrowse/`
(`baseUrl: /msgbrowse/`).

## Ground rules for content

- Verify every command and flag against the repo before writing:
  `internal/cli/*.go` (cobra `Use:`/`Flags()`), `internal/config/config.go`
  (viper keys), plus `README.md`, `SECURITY.md`, `ARCHITECTURE.md`.
  Real commands as of this scaffold: `export`, `import`, `signal-import`,
  `imessage-import`, `sync`, `serve`, `doctor`, `facts`, `embed`, `mcp`,
  `journal`, `media`, `watch`, `version`.
- Hand-authored pages are MDX: escape literal `{`, `}`, and `<` outside code
  fences (the escaping transform only runs on generated ADR/spec content).

## Known caveats

- "Edit this page" links point at `https://github.com/joestump/msgbrowse/tree/main/<generated path>`;
  for generated ADR/spec pages the true sources are `docs/adr/` and
  `docs/openspec/specs/`. Cosmetic — fix later with an `editUrl` function if
  it bothers anyone.
- The spec cross-reference mapping (`src/data/spec-mapping.json`) is
  prefix-based and this repo uses a single `SPEC-` prefix for all specs, so
  bare `SPEC-XXXX` mentions in prose all link to one spec page. Explicit
  markdown links between specs/ADRs (which the source artifacts already use)
  resolve correctly.
- ADRs in this repo keep status/date as MADR bullet lists, not YAML
  frontmatter, so the status badges on ADR pages read `UNKNOWN` and the graph
  page has no ADR nodes/edges. Adding frontmatter to the source ADRs would
  light these up.
